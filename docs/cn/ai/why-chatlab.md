# 为什么选择 ChatLab

聊天记录里藏着很多信息：你和某个人聊过什么、一个群最近在讨论什么、谁最活跃、什么时候最热闹。但导出的聊天文件动辄几万条消息，人翻不过来，直接丢给 AI 也不好用。

ChatLab 先把聊天记录整理成结构化数据，提问时只取出真正相关的部分交给 AI——**更快、更省，也更安全**。

## 为什么不直接把聊天文件发给 AI？

把整份文件直接丢给 AI 分析，通常会遇到三个问题：

- **太大读不完**：几万条消息超出模型一次能处理的长度，被截断后容易漏掉关键内容。
- **夹带无关内容**：你只想问一件小事，却把大量无关的聊天也一并发了出去。
- **隐私没法把关**：聊天里可能混着密码、银行卡号、身份证、住址等敏感信息，整份上传可能会把这些内容一并**暴露给外部 AI 服务商**。

总而言之，把整份聊天记录直接交给 AI，**Token 花得更多，结果未必更准，隐私风险却更高**。

多数问题真正需要的，只是其中一小部分记录。

## ChatLab 怎么解决

- **先整理，再分析**：把不同平台的聊天记录统一成会话、成员、消息，结构清晰、可检索。
- **专用工具**：提供搜索、上下文、统计分析、成员画像、图表绘制、RAG 检索等能力，AI 会根据问题按需调用，只取当前分析需要的数据。
- **隐私防护**：手机号、身份证号、银行卡号、邮箱、API Key 等敏感信息，会移除并替换成占位符，再交给 AI。
- **本地优先**：聊天数据库保存在你自己的电脑上，只有当前问题需要的少量结果，才会经过处理后交给 AI。

## 如何使用ChatLab提供的AI能力？

- **[内置 AI](/cn/ai/chatlab-ai)**：在 ChatLab 应用内配置一个 AI 模型，直接提问和分析。适合想开箱即用的人。
- **[外部 AI Agent](/cn/ai/external-agent)**：安装 ChatLab CLI 和官方 Skill，让 Codex、Claude Code 等 Agent 查询你的聊天记录。适合已有常用 Agent、想复用现有工作流的人。

两种方式用的都是已导入 ChatLab 的记录，**默认都经过数据脱敏和黑名单过滤**。
