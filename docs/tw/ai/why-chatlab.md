# 為什麼選擇 ChatLab

聊天記錄裡藏著很多資訊：你和某個人聊過什麼、一個群組最近在討論什麼、誰最活躍、什麼時候最熱鬧。但匯出的聊天檔案動輒幾萬則訊息，人翻不完，直接丟給 AI 也不好用。

ChatLab 會先把聊天記錄整理成結構化資料，提問時只取出真正相關的部分交給 AI——**更快、更省，也更安全**。

## 為什麼不直接把聊天檔案傳給 AI？

把整份檔案直接丟給 AI 分析，通常會遇到三個問題：

- **太大讀不完**：幾萬則訊息超出模型一次能處理的長度，被截斷後容易遺漏重要內容。
- **夾帶無關內容**：你只想問一件小事，卻把大量無關的聊天也一併傳了出去。
- **隱私沒法把關**：聊天裡可能混著密碼、銀行卡號、身分證、住址等敏感資訊，整份上傳可能會把這些內容一併**暴露給外部 AI 服務商**。

總而言之，把整份聊天記錄直接交給 AI，**Token 花得更多，結果未必更準，隱私風險卻更高**。

多數問題真正需要的，只是其中一小部分記錄。

## ChatLab 怎麼解決

- **先整理，再分析**：把不同平台的聊天記錄統一成工作階段、成員、訊息，結構清晰、可檢索。
- **專用工具**：提供搜尋、上下文、統計分析、成員畫像、圖表繪製、RAG 檢索等能力，AI 會依問題按需調用，只取目前分析需要的資料。
- **隱私防護**：電話號碼、身分證號、銀行卡號、電子郵件、API Key 等敏感資訊，會移除並替換成佔位符，再交給 AI。
- **本機優先**：聊天資料庫保存在你自己的電腦上，只有目前問題需要的少量結果，才會經過處理後交給 AI。

## 如何使用 ChatLab 提供的 AI 能力？

- **[內建 AI](/tw/ai/chatlab-ai)**：在 ChatLab 應用程式內配置一個 AI 模型，直接提問和分析。適合想開箱即用的人。
- **[外部 AI Agent](/ai/external-agent)（英文）**：安裝 ChatLab CLI 和官方 Skill，讓 Codex、Claude Code 等 Agent 查詢你的聊天記錄。適合已有常用 Agent、想沿用現有工作流程的人。

兩種方式用的都是已匯入 ChatLab 的記錄，**預設都經過資料脫敏和黑名單過濾**。
